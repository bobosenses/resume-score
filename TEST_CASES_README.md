# 简历打分测试用例说明

## 文件列表

1. **test_cases.json** - 测试数据集（22KB）
   - 包含 3 份不同背景的候选人简历
   - 完整的职位描述（JD）
   - 详细的打分规则（100分制）

2. **test_scoring.py** - 测试脚本（13KB）
   - 自动化批量测试
   - 支持 Web API 和 vLLM API
   - 支持思考模式对比测试

## 使用方法

### 1. 运行全部测试（关闭思考模式）

```bash
python3 /root/vLLM/test_scoring.py --api web
```

### 2. 运行全部测试（开启思考模式）

```bash
python3 /root/vLLM/test_scoring.py --api web --thinking
```

### 3. 测试指定简历

```bash
# 只测试第1份简历（李明远）
python3 /root/vLLM/test_scoring.py --api web --resume-ids 1

# 测试第1和第3份简历
python3 /root/vLLM/test_scoring.py --api web --resume-ids 1 3
```

### 4. 使用 vLLM API（端口 8000）

```bash
python3 /root/vLLM/test_scoring.py --api vllm --thinking
```

### 5. 对比测试（思考模式开启 vs 关闭）

```bash
python3 /root/vLLM/test_scoring.py --compare
```

## 评分维度说明

### 一、硬性条件匹配（30分）
1. **学历与院校**（6分）：985/211 + 计算机/AI相关专业
2. **工作年限与职级**（6分）：8年+ + 总监级别
3. **AI相关经验**（10分）：大模型产品经验（**核心要求**）
4. **数据能力**（8分）：SQL + A/B实验 + 数据评估体系

### 二、软性能力评估（25分）
1. **产品方法论**（8分）：系统化方法论 + 具体应用案例
2. **跨团队协作**（9分）：20人+跨职能团队 + 算法团队协调
3. **创新与洞察**（8分）：行业级创新产品 + 技术趋势洞察

### 三、成果与影响力（30分）
1. **量化业务成果**（18分）：DAU/营收/GMV等关键指标增长
2. **产品影响力范围**（12分）：千万级用户产品 + 行业影响力

### 四、发展潜力与匹配度（15分）
1. **成长速度**（7分）：快速晋升 + 带人经验
2. **综合素质**（8分）：技术背景 + 管理 + 国际视野

## 思考模式对比

### 关闭思考模式（默认）
- **耗时**：3-5秒
- **输出**：直接给出评分结果
- **适用场景**：快速筛选大量简历

### 开启思考模式
- **耗时**：10-20秒
- **输出**：包含详细推理过程（`<think>...</think>`）
- **适用场景**：重要候选人深度评估


## API 调用示例

### 使用 test_cases.json 中的数据

```python
import json
import requests

# 加载测试数据
with open("/root/vLLM/test_cases.json", "r") as f:
    data = json.load(f)

resume = data["test_cases"]["resumes"][0]["content"]
jd = data["test_cases"]["jd"]["content"]
rules = data["test_cases"]["scoring_rules"]["content"]

# 构建 prompt
prompt = f"""你是一位资深HR和技术面试官，严格按照打分规则评估候选人。

=== 候选人简历 ===
{resume}

=== 职位描述(JD) ===
{jd}

=== 打分规则 ===
{rules}"""

# 调用 API
resp = requests.post(
    "http://8.163.39.222:3000/api/generate",
    json={
        "prompt": prompt,
        "thinking": True,
        "options": {"temperature": 0.1, "num_predict": 2000}
    }
)

# 解析结果
for line in resp.text.strip().split("\n"):
    chunk = json.loads(line)
    if chunk.get("response"):
        print(chunk["response"], end="")
```

## 注意事项

1. **Token 消耗**：每份简历测试约消耗 8000-10000 tokens（输入）+ 1000-2000 tokens（输出）
2. **并发限制**：建议串行测试，避免 GPU 显存溢出
3. **温度设置**：评分任务建议使用低温度（0.1-0.3），保证结果一致性
4. **思考模式**：会增加 3-5 倍 token 消耗和响应时间，仅重要候选人使用

## 常见问题

**Q: 为什么同一份简历多次测试分数不同？**  
A: LLM 输出有随机性，即使 temperature=0.1 也会有轻微波动（±5分属正常范围）。

**Q: 思考模式的分数为什么通常更低？**  
A: 思考模式会进行更深入的推理，更容易发现简历中的问题和不足，评分更严格。

**Q: 如何批量测试多份真实简历？**  
A: 将简历内容添加到 `test_cases.json` 的 `resumes` 数组中，然后运行测试脚本。
